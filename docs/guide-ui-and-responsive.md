# Руководство по UI и responsive-вёрстке

Единые требования к вёрстке: mobile-first и отсутствие полного скролла страницы. Детали правил: [UI.mdc](../.cursor/rules/UI.mdc).

## Mobile-first

Стили по умолчанию пишем для малого экрана. Улучшения для планшетов и десктопов — через media queries с `min-width` от меньшего к большему. Не используйте `max-width` как основу: сначала база для мобильного, потом добавление для больших экранов.

- **Viewport:** в `index.html` должен быть `<meta name="viewport" content="width=device-width, initial-scale=1">`. Горизонтального скролла быть не должно (контент и отступы вписываются в ширину экрана).
- **Breakpoints:** если в проекте есть токены (например в `src/app/styles/tokens.css`) — используйте их. Иначе ориентир: 768px (планшет), 1024px (десктоп). Порядок в коде: сначала стили без media, затем `@media (min-width: 768px)`, затем `@media (min-width: 1024px)`.
- **Чеклист для новых страниц и компонентов:** перед сдачей проверьте отображение на узком viewport (320–390px); только после этого — на широком.

## Без скролла всей страницы

Цель: одна логическая страница по высоте равна одному viewport. Пользователь не скроллит «всю страницу» — скролл только внутри выделенной контентной области (например основной блок под шапкой).

Если контента больше высоты экрана, скролл должен быть у внутреннего блока (`main`, `.page__content` или аналог), а не у `body` или корневого контейнера страницы. Шапка и футер остаются на месте; растёт и скроллится только средняя часть.

**Техника:**

- Корневой контейнер страницы/лейаута: высота на весь экран и колонка flex, например `height: 100dvh` (или `100vh`) и `display: flex; flex-direction: column`.
- Шапка и футер — без `flex-grow`, фиксированной высоты или авто.
- Контентная область: `flex: 1; min-height: 0; overflow-y: auto`. Тогда только она скроллится.

Пример структуры (имена классов условные):

```html
<div class="layout" style="height: 100dvh; display: flex; flex-direction: column;">
  <header class="layout__header">...</header>
  <main class="layout__content" style="flex: 1; min-height: 0; overflow-y: auto;">
    <!-- контент страницы -->
  </main>
  <footer class="layout__footer">...</footer>
</div>
```

В Vue-лейаутах ту же схему можно задать через классы в корне лейаута и слот для контента.
